#第一段
我希望用vide coding的方式制作一个小游戏，核心玩法是用鼠标能牵引一只爬行类的动物在屏幕上任意行走和猎食。我希望这个动物能表现出流畅的多关节跟随系统的视觉表现，核心模型的描述如下：
爬行动物的脊椎骨骼一条由N个圆形节点构成的链式结构（例如20个节点）构成，节点之间通过线段连接，首个节点的位置在最大速度内实时跟随鼠标光标，后续的每一个跟随子节点必须严格维持与前驱节点的固定距离（比如30px）。在每一帧更新位置的时候，通过计算向量差将子节点的位置修正到以其前驱节点为圆心，固定距离为半径的圆周上，方向取自子节点当前位置指向前驱节点的方向，从而实现自然的“拖拽”+“蛇形”跟随效果。在视觉呈现上，背景为深色，节点用实心圆表示，连接线用白色的线条，整体需要保证交互流畅，代码结构清晰以便调整节点数和段长。
爬行动物长有4只脚，每只脚与脊椎相连，有两个关节，对称分布在身体两侧。当脊椎向前运动一段距离的时候，脚会自动带动到新的位置，然后利用IK反关节求解器，计算关节对应的运动角度以描绘出自然的爬行动作。
最后我们要利用矢量动画，给这套骨骼系统套上外皮肤，使之成为一个逼真的动画。
整个程序使用web H5技术开发，轻量级应用。
请你评估这套开发系统的技术难度和关键实现算法，并帮我解决其中可能存在的问题。

#第二段
现在折返运动的时候会穿越自己身体，我们要避免自碰撞，所以需要让关节连线在可能发生自穿越的时候会寻路到最近的绕行线路，以避免自碰撞。

#第三段
动物的骨骼关节都有一定的最小弯折角度，我们要给身体不同位置的关节一些角度约束，以避免身体过度柔软，让运动表现更自然。

#第四段
爬行动物运动的时候，身体会有自动的蛇形波动，我们需要让目前的关节系统在运动的时候，从胸部以后产生自然的蛇形波动效果，并逐渐向尾部传递并放大。

#第五段
在窗口下部增加一个参数区，请将所有的运动参数数值罗列在参数区，并可以用箭头控件微调。参数设置可以保存成Json文件。
