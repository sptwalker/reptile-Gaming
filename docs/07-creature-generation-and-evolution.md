## 2. 幼体诞生随机生成规则

### 2.1 基础随机外观流程

生成幼体前先执行一次完整随机外观：

1. 随机外观模块中的所有 `range` 参数。
2. 随机头型：从 `ellipse`、`triangle`、`inverted_triangle`、`shovel`、`crescent`、`fan`、`semicircle`、`diamond` 中选取。
3. 头部大小不得小于 `minHeadScaleForBodyWidth(bodyWidth)`。
4. 应用生物力学随机约束。
5. 棘刺初始随机：`50%` 无棘刺，`50%` 生成 `6~40` 根；棘刺长度 `0.35~2.2`。
6. 头部、躯干、尾部、眼睛、花纹颜色随机。
7. 四肢颜色按“四肢随机色规则”生成。
8. 花纹类型从 6 种中随机。
9. 清空隐藏基因。
10. 应用参数并同步颜色控件。

### 2.2 幼体覆盖规则

幼体生成会在基础随机外观后覆盖为幼体特征：

| 项目 | 幼体规则 |
| --- | --- |
| 生命周期 | `lifeStage = 0`。 |
| 脊柱节点 | 随机 `12~17`。 |
| 躯干长度 | 随机 `0.65~1.19`。 |
| 生物力学 | 在幼体体长重设后再次应用一次生物力学规则。 |
| 上肢长度 | 当前值乘以 `0.72`，钳制到 `20~58`。 |
| 下肢长度 | 当前值乘以 `0.72`，钳制到 `18~54`。 |
| 上下肢间隔 | 随机 `1.35~1.45`，幼体肢体间距更大。 |
| 头部大小 | `minHeadScaleForBodyWidth(bodyWidth) * 1.08`，并钳制到 `0.55~2.2`。 |
| 棘刺数量 | 固定 `0`，幼体出生无棘刺。 |
| 棘刺长度 | 固定 `1`，作为后续生长基准。 |
| 饱和倍率 | 随机 `0.45~0.95`，幼体更低饱和。 |
| 花纹复杂度 | 固定 `1`，幼体花纹简单。 |
| 明度 | 随机 `16~49`。 |
| 四肢颜色 | 饱和度 `18~44`、明度 `16~min(42, lightness+2)`，并优先接近躯干色。 |

### 2.3 生物力学随机约束

随机外观和幼体生成都会通过体长、体宽、体量修正运动与关节：

- `bodyLength = segmentWidth * (spineNodes / 22)`。
- `bodyBulk = bodyLength * bodyWidth`。
- 四肢粗细会随体量提高，目标为 `0.65 + bodyBulk * 0.35`，钳制 `0.55~1.8`。
- 四肢总长度会按体长目标比例修正，目标比例钳制 `0.75~1.25`。
- 体量越重，移动速度和追逐倍率越低。
- 短腿会降低步幅、提高迈步速度。
- 身体越宽，蛇形摆幅降低，碰撞边距提高。
- 体型越重，转向强度降低，停顿概率和停顿时长提高。
- 关节弯折按体宽/体量降低，但保留最低弯折下限，避免僵硬。

## 3. 蜕变升级规则

### 3.1 测试页视觉蜕变规则

测试页最多蜕变 4 次。每次点击“蜕变升级”从当前阶段进入下一阶段。

| 目标阶段 | 体宽倍率 | 躯干长度倍率 | 头部倍率 | 四肢长度倍率 | 四肢粗细倍率 | 饱和倍率 | 速度倍率 | 步幅倍率 | 上下肢间隔 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 少年 1 | 1.08 | 1.08 | 1.08 | 1.08 | 1.08 | 1.04 | 0.96 | 1.06 | 1.25 |
| 青年 2 | 1.08 | 1.08 | 1.08 | 1.08 | 1.08 | 1.05 | 0.95 | 1.07 | 1.12 |
| 成年 3 | 1.09 | 1.09 | 1.08 | 1.09 | 1.09 | 1.05 | 0.94 | 1.08 | 1.00 |
| 完全体 4 | 1.09 | 1.09 | 1.08 | 1.09 | 1.09 | 1.06 | 0.93 | 1.08 | 1.00 |

每次测试页蜕变还会执行：

- 脊柱节点随机增加 `+2` 或 `+3`，钳制 `12~36`。
- 头部大小保持不低于当前体宽所需最低值。
- 花纹复杂度 `+1`，钳制 `1~6`。
- 移动速度按阶段倍率略降，体现体型变大后的沉重感。
- 腿频同步为移动速度。
- 明度随机增加 `+1~+4`，钳制 `16~58`。
- `headColor`、`bodyColor`、`limbColor`、`tailColor` 如为可解析 HSL/HEX，会同步提高明度。
- 蜕变后重新应用参数、重建面板并同步颜色控件。

### 3.2 测试页棘刺蜕变规则

| 当前棘刺状态 | 蜕变结果 |
| --- | --- |
| 无棘刺 | `20%` 概率新生棘刺。新生数量 `4~10`，长度 `0.45~0.8`。 |
| 已有棘刺 | 数量随机增加 `+2~+6`，钳制 `1~40`；长度随机增加 `+0.08~+0.28`，钳制 `0.35~2.2`。 |

### 3.3 头部棘刺渲染规则

- 棘刺渲染由 `client/js/lizard-renderer.js` 执行。
- 棘刺分布大致为：身体 `55%`、四肢 `25%`、头部为剩余数量。
- 头部棘刺不再使用固定坐标，而是通过当前 `headShape` 动态计算头部轮廓半宽与锚点。
- 支持 8 种头型轮廓适配，避免三角形、月牙形、扇形等特殊头型出现棘刺悬空。
- 当头型变化时，头部棘刺锚点每帧按新头型重新计算并同步调整。

### 3.4 正式后端蜕变校验规则

正式接口 `evolvePet(uid, petId, ip)` 校验链：

1. 宠物存在。
2. 应用时间衰减。
3. 阶段未超过品质允许上限。
4. 等级满足目标阶段要求。
5. 属性数据存在。
6. 成年到完全体检查品质要求。
7. 目标阶段若有属性门槛，则任意六维总和需达标。
8. 体力满足 `50`。
9. 金币满足目标阶段消耗。

| 目标阶段 | 阶段变化 | 等级要求 | 金币消耗 | 属性要求 | 品质要求 |
| --- | --- | ---: | ---: | --- | --- |
| 1 | 幼体→少年 | 10 | 100 | 无 | 无 |
| 2 | 少年→青年 | 20 | 200 | 任意属性总和 ≥30 | 无 |
| 3 | 青年→成年 | 30 | 300 | 无 | 无 |
| 4 | 成年→完全体 | 50 | 500 | 任意属性总和 ≥60 | 品质 ≥ 优秀 |

通用消耗与成长：

- 体力消耗：`50`。
- 阶段 `+1`。
- 体力上限 `+20`。
- 饱食上限 `+10`。
- 六维基础属性 `str/agi/vit/int/per/cha` 全部 `+3`。
- 重新计算衍生属性。
- 返回新等级上限：阶段 0/1/2/3/4 分别为 `10/20/30/50/100`。

### 3.5 正式后端技能解锁规则

| 目标阶段 | 技能池 |
| --- | --- |
| 1 | `scratch`、`tail_whip` |
| 2 | `camouflage`、`venom_spit`、`iron_hide` |
| 3 | `dragon_rush`、`regen`、`predator_eye` |
| 4 | `dragon_rush`、`regen`、`predator_eye` |

- 每次蜕变从目标阶段技能池中过滤已拥有技能。
- 若仍有候选技能：
  - 品质 ≥ 稀有：保底解锁 1 个。
  - 品质 < 稀有：`60%` 概率解锁 1 个。
- 解锁技能默认 `skill_level = 1`，装备到下一个可用技能栏。
- 技能栏上限：普通 `2`，优秀 `3`，稀有 `3`，史诗 `4`，传说 `4`。

### 3.6 正式后端外观持久化蜕变规则

正式蜕变会把外观成长写入 `pet.body_seed`：

| 字段 | 规则 |
| --- | --- |
| `evolveSpineBonus` | 当前值存在则沿用，否则默认为 `pet.stage * 2`；每次蜕变随机增加 `+2` 或 `+3`。 |
| `legGapRatio` | 目标阶段 1 为 `1.25`，2 为 `1.12`，3/4 为 `1.0`。 |
| `lightness` | 每次增加 `+1~+4`，钳制 `16~58`。 |
| `bodyLightness` | 若存在，同步增加 `+1~+4`，钳制 `20~80`。 |
| 固定色 | `headColor/bodyColor/limbColor/tailColor` 若为 HSL，会同步提高明度。 |
| `spineCount` | 无棘刺时 `20%` 概率新生 `4~10`；已有棘刺时增加 `+2~+6`，正式上限 `80`。 |
| `spineLength` | 新生时 `0.45~0.8`；已有棘刺时增加 `+0.08~+0.28`，正式上限 `3`。 |

正式蜕变返回与日志会记录：

- `spine_bonus_gain`
- `spine_bonus_total`
- `leg_gap_ratio`
- `lightness_gain`
- `lightness_after`
- `spine_mutation`
- `spine_count`
- `spine_length`
- `skill_unlocked`

### 3.7 宠物详情渲染输出规则

`getPetDetail()` 会把正式数据转为前端可渲染参数：

| 渲染字段 | 来源/公式 |
| --- | --- |
| `bodyWidth` | 基础值 × 力量加成 × 阶段倍率。 |
| `headScale` | 基础值 × 力量加成 × 阶段倍率。 |
| `moveSpeed` | 基础值 × 敏捷加成。 |
| `legFrequency` | 基础值 × 敏捷加成。 |
| `spineNodes` | 基础脊柱节点 + `floor(vit/10)` + `evolveSpineBonus`。 |
| `segmentWidth` | 基础值 × 体质加成 × 阶段倍率。 |
| `fovAngle` | 基础值 × 智力加成。 |
